# CloverleafTrack — Testing Guide

## Stack

| Package | Role |
|---|---|
| xUnit 2.x | Test runner and assertion primitives |
| FluentAssertions 8.x | Readable assertion syntax (`.Should().Be(...)`) |
| Moq 4.x | Mock/stub repository interfaces |
| coverlet | Code coverage collection (configured in `.csproj`) |

Run the full suite:

```bash
dotnet test CloverleafTrack.Tests/CloverleafTrack.Tests.csproj
```

## Model Tests

### MeetTests

`Meet` is the only domain model with non-trivial computed properties.

**`Meet.Slug`** — generated by `SlugHelper` (Slugify NuGet package) from `Meet.Name`:
- Lowercases and hyphenates words: `"Spring Invitational"` → `"spring-invitational"`
- **SlugHelper keeps periods** but strips apostrophes: `"St. Mary's"` → `"st.-marys"` (not `"st-marys"`)
- Used as the URL identifier for every meet page

**`Meet.ResultsUrl`** — derived as `/meets/{Slug}`:
- Always starts with `/meets/`
- The URL segment after `/meets/` is always the slug

## ViewModel Tests

ViewModels are tested for any computed property or derived value that contains meaningful logic. Simple string concatenations (`FullName = $"{First} {Last}"`) are not tested independently since they're trivially verified through service-layer tests.

### Admin ViewModels

**`SeasonProgressViewModel.PercentComplete`**
- Returns `0` when `TotalMeets == 0` (divide-by-zero guard)
- Uses **integer division** (`EnteredMeets * 100 / TotalMeets`) — truncates, does not round
  - 1 of 3 = 33 (not 34); 2 of 3 = 66 (not 67)

**`LocationOptionViewModel.DisplayText`**
- Returns `"Name (City, State)"` when both city and state are non-empty
- Falls back to just `Name` when either city or state is empty/whitespace

**`AthleteOptionViewModel.DisplayText`** — `"LastName, FirstName (Year)"` (last name first, comma-separated)

**`EventOptionViewModel`**
- `DisplayText` — `"EventName (Gender)"`
- `CategoryName` — `EventCategory.ToString()` when set; `"Other"` when null

**`MeetOptionViewModel.DisplayText`** — `"Name - MMM d, yyyy (Environment)"` (e.g. `"Spring Invitational - Mar 15, 2024 (Outdoor)"`)

### Athlete ViewModels

**`IndividualPerformanceViewModel`** and **`PersonalRecordViewModel`** share the same relay parsing pattern:

| Property | Behaviour |
|---|---|
| `IsRelay` | `true` when `RelayAthletes != null`, `false` when null |
| `RelayMembers` | `RelayAthletes?.Split("|~|") ?? Array.Empty<string>()` |

### Leaderboard ViewModels

**`LeaderboardEventViewModel.RelayMembers`** — uses `string.IsNullOrEmpty(RelayName)` as the guard (different from athlete VMs which use a null check). This is intentional: `RelayName` is initialized to `string.Empty`, while `RelayAthletes` is nullable. Both null and empty-string `RelayName` return an empty list.

### Meets ViewModels

**`MeetListItemViewModel.IsUpcoming`** — `Date > DateTime.Now`. Past and far-future dates both covered in tests.

### Seasons ViewModels

**`SeasonCardViewModel`**:
- `IndoorSchoolRecordCount` — `IndoorSchoolRecords?.Count ?? 0` (null-safe)
- `OutdoorSchoolRecordCount` — `OutdoorSchoolRecords?.Count ?? 0` (null-safe)
- `StatusBadge` — `Status.ToString()` for all `SeasonStatus` values

## Service Tests

Service tests follow a common pattern:

```csharp
// 1. Create mock repository
var mockRepo = new Mock<IFooRepository>();

// 2. Configure what the mock returns
mockRepo.Setup(r => r.GetSomethingAsync(...)).ReturnsAsync(...);

// 3. Construct the real service with the mock
var service = new FooService(mockRepo.Object);

// 4. Call the method under test
var result = await service.DoSomethingAsync(...);

// 5. Assert with FluentAssertions
result.Should().NotBeNull();
result.SomeProperty.Should().Be(expectedValue);
```

For tests that need many setup calls (e.g. `MeetServiceTests`), the `SetupMeetDetails` helper centralises the three-call setup so individual tests only vary the data.

### AthleteServiceTests

Covers two main areas:

**Roster grouping** (`GetActiveAthletesGroupedByEventCategoryAsync`, `GetFormerAthletesGroupedByGraduationYearAsync`):
- Only active athletes appear in the active roster; only inactive athletes appear in the former roster.
- Best distance is `MAX(DistanceInches)`; best time is `MIN(TimeSeconds)` — both formatted via `PerformanceFormatHelper`.

**Athlete detail page** (`GetAthleteDetailsAsync`):
- Returns `null` when the slug is not found.
- Individual PRs use the `PersonalBest = true` flag (reliable for individuals).
- Relay PRs use best-per-event regardless of the `PersonalBest` flag (unreliable for relays — see BRAIN.md C8).
- `TotalPRs` counts only individual performances where `PersonalBest = true`.
- `TotalSchoolRecords` = individual `SchoolRecord = true` rows + relay events where `AllTimeRank == 1` (distinct by EventId).
- `IsSchoolRecord` on a relay `PersonalRecordViewModel` is set when `AllTimeRank == 1`, not from the `SchoolRecord` flag.
- Seasons are ordered most-recent-first using `SeasonStartDate` from the DTO (not string sort on season name).
- Relay team members are parsed from the `|~|`-separated `RelayAthletes` string.

### LeaderboardServiceTests

Covers `GetLeaderboardAsync` and `GetLeaderboardDetailsAsync`:

**Partitioning:**
- `Gender.Male` → `BoysOutdoor/IndoorCategories`
- `Gender.Female` → `GirlsOutdoor/IndoorCategories`
- `Gender.Mixed` → `MixedOutdoor/IndoorCategories` only — not in Boys or Girls
- `Environment.Outdoor` → Outdoor lists; `Environment.Indoor` → Indoor lists

**Category grouping:**
- Non-relay events are grouped by `EventCategory` (Sprints, Distance, Hurdles, Jumps, Throws).
- `EventType.RunningRelay` → "Running Relays" category
- `EventType.JumpRelay` → "Jump Relays" category
- `EventType.ThrowsRelay` → "Throw Relays" category
- Category order: Sprints → Distance → Hurdles → Running Relays → Jumps → Jump Relays → Throws → Throw Relays

**Details page** (`GetLeaderboardDetailsAsync`):
- Returns `null` when no performances exist for the event key.
- `AllPerformances` includes every row returned by the repository.
- `PersonalRecordsOnly` filters to one row per athlete (best first), excludes relay rows (`AthleteId == null`).
- `IsRelayEvent` is `true` when any performance row has `AthleteId == null`.

### MeetServiceTests

Covers `GetMeetDetailsAsync` and `GetMeetsIndexAsync`:

**Meet details:**
- Returns `null` when the slug is not found.
- Surfaces location name/city/state from the `Location` navigation property.
- Counts `PersonalBest = true` rows for `TotalPRs`, `SchoolRecord = true` rows for `TotalSchoolRecords`.
- Splits performances into `BoysEvents`, `GirlsEvents`, `MixedEvents` by `EventGender`.
- `UniqueAthletes` passes through from the repository call.

**Event ordering within a meet** (via `BuildOrderedEventGroups`):
- Sprints → Distance → Hurdles → Running Relays → Jumps → Jump Relays → Throws → Throw Relays
- Within each category, events are ordered by `EventSortOrder` ascending.

**Meets index** (`GetMeetsIndexAsync`):
- `TotalMeets` is the flat count of all meets.
- Meets are grouped by season; `TotalSeasons` is the distinct season count.
- Seasons are ordered descending by `StartDate` (most recent first).
- Meets within each season are ordered ascending by `Date` (oldest first).

### SeasonServiceTests

- `GetCurrentSeasonAsync()` returns `EndDate.Year` of the season where `IsCurrentSeason = true` — **not** the season's database `Id`.
- Throws `InvalidOperationException("No current season found.")` when no season has `IsCurrentSeason = true`.

## Utility Tests

### PerformanceFormatHelperTests

`PerformanceFormatHelper` (`CloverleafTrack.Web/Utilities/PerformanceFormatHelper.cs`) is a static helper class. Tests cover each public method.

**`ParseTime(string?)`** → `double?`

| Input | Result |
|---|---|
| `null` / `""` / whitespace | `null` |
| `"11.24"` | `11.24` |
| `"11.24s"` | `11.24` |
| `"1:23.45"` | `83.45` |
| `"1m23.45s"` | `83.45` |
| `"1:00.00"` | `60.0` |
| non-numeric | `null` |

**`FormatTime(double)`** → `string`

| Input | Result |
|---|---|
| `11.24` | `"11.24"` |
| `60.0` | `"1:00.00"` |
| `83.45` | `"1:23.45"` |
| `65.5` | `"1:05.50"` (leading zero on seconds) |

**`ParseDistance(string?)`** → `double?` (total inches)

| Input | Result |
|---|---|
| `null` / `""` | `null` |
| `"19'4"` | `232.0` |
| `"19'4.5"` | `232.5` |
| `"19-04"` | `232.0` |
| `"234.5"` | `234.5` |
| `"19 feet 4 inches"` | `232.0` |
| non-numeric | `null` |

**`FormatDistance(double)`** → `string`

Canonical output format is always `"F' I\""` with a space after the apostrophe.

| Input (inches) | Result |
|---|---|
| `232.0` | `"19' 4\""` |
| `232.5` | `"19' 4.5\""` |
| `144.0` | `"12' 0\""` |

**Note on round-trips:** `ParseDistance` accepts compact input (`"19'4"`) but `FormatDistance` always outputs the canonical spaced form (`"19' 4\""`). These two formats are not identical — round-trip tests assert against the canonical formatted output, not the original input string.

**`FormatPerformance(double? timeSeconds, double? distanceInches)`** → `string`
- Time present → `FormatTime(timeSeconds.Value)`
- Distance present → `FormatDistance(distanceInches.Value)`
- Both null → `"N/A"`

**`FormatImprovement(double? newTime, double? prevTime, double? newDist, double? prevDist)`** → `string`
- Time improved (new < prev): `"-0.50s faster"`
- Time worsened (new > prev): `"+0.50s slower"`
- Distance improved (new > prev): `"+1' 0\" farther"`
- Distance worsened (new < prev): `"-1' 0\" shorter"`
- All null: `""`

## Key Invariants Tested

The most important correctness invariants covered by the tests (non-obvious, see BRAIN.md for full context):

1. **Relay `PersonalBest` flag is unreliable.** `AthleteService` must use best-per-event for relay PRs, not the flag. Test: `GetAthleteDetailsAsync_RelayPRs_UseBestPerEvent_NotPersonalBestFlag`.

2. **Relay `SchoolRecord` flag is unreliable.** Use `AllTimeRank == 1` as the school record proxy for relay performances. Test: `GetAthleteDetailsAsync_TotalSchoolRecords_IncludesRelayEventsWhereAllTimeRankIsOne`.

3. **`TotalPRs` counts only individual performances.** Relay `PersonalBest = true` must not inflate the PR count. Test: `GetAthleteDetailsAsync_TotalPRs_CountsOnlyIndividualPerformances`.

4. **Mixed relay performances must not appear in Boys or Girls leaderboard categories.** Test: `GetLeaderboardAsync_PlacesMixedPerformances_InMixedCategories_NotBoysOrGirls`.

5. **Season ordering uses `SeasonStartDate`, not season name string.** Test: `GetAthleteDetailsAsync_Seasons_OrderedMostRecentFirst`.

6. **Meets within a season are ordered oldest-first.** Test: `GetMeetsIndexAsync_OrdersMeetsWithinSeason_AscendingByDate`.
