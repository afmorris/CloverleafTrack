Mockup D — Multi-Column Options

Four layout approaches, same data

The page uses max-w-7xl (1280px) — plenty of horizontal room. Each option below is a live interactive demo. Scroll through all four to compare.

Option 1 — Table left · Chart right
The existing performance rows stay exactly as-is on the left. The chart lives on the right and updates when you click an event tab. Best for athletes with 2–4 events. Feels like a natural extension of the existing design.

Option 2 — Side-by-side charts (2-up grid)
Dedicated chart section with two full charts displayed simultaneously. No tabs — both events visible at once. Great for direct comparison ("did Shot Put and Discus improvement track together?"). Collapses to 1-col on narrow screens.

Works best when: athlete competes in 2–4 events. At 5+ events the grid becomes overwhelming; you'd want to cap it at 2-up and add a "show more" control. The comparison value is high — you can immediately see whether both events improved in sync. Best for multi-event athletes
Option 3 — 3-column mini-chart grid
Compact cards in a 3-column grid. Less chart detail — the point is a quick overview of all events at once, not deep analysis. The chart here is tiny and best used to show the trend shape (flat, improving, inconsistent) rather than exact values. Good for sprinters who compete in 5–7 events.

Option 4 — Hero chart (best event) · Secondary grid below
The athlete's strongest event (best all-time rank, or the one they've competed in most) gets a full-width hero chart with full detail. Secondary events sit below in a 2-column grid at smaller size. Naturally prioritizes the event that matters most.
